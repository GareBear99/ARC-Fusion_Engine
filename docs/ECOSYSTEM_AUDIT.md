## Legend

- **Public**: `public` · `private` · `empty` (exists on GitHub but has no commits) · `archived`.
- **License**: SPDX id reported by GitHub (e.g. `MIT`, `Apache-2.0`), or `none`.
- **README**: `Y` if `README.md` exists at repo root, `N` if not, `-` for empty repos.
- **FUNDING**: `Y` if `.github/FUNDING.yml` exists, `N` if not. FUNDING drives the GitHub "Sponsor" button.
- **main protected**: whether `branches/main/protection` returns a protection rule. `n/a` when the default branch is not `main`.
- **Pages**: `built` / `building` / `errored` / `no` (GitHub Pages status).

## Audit rules of thumb

1. Every public repo should have a `README.md`, a `LICENSE`, and (for solo-author work) a `FUNDING.yml`.
2. `main` on primary-development repos should block force-push and deletion (already applied to the core loop).
3. Empty repos are OK as reserved names; they should not be linked from user-facing docs without a "coming soon" flag.
4. Any repo with open issues > 0 merits a triage pass; with open issues > 10 merits an SLA.
5. Pages should be `built` on any repo the user links from the Portfolio or another public surface.
